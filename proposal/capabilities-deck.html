The Problem

A capable system that can't safely move forward

  • The Reports module has known, unresolved defects from the previous build.
  • Operations has no way to turn stock and demand data into a purchasing decision.
  • Zero automated test coverage means IT won't approve further changes.
  • The dashboard is stuck: functional today, unable to safely evolve.
2 / 12
What We Found

The gaps are specific, not vague

Reports.vue

The one view still on the old code pattern, bypassing the shared data layer, with leftover debug logging and no translation support.

Restocking

Stock and demand data already exist in the system. A model for purchase orders was even started, but never connected.

Testing

No automated coverage exists anywhere in the application today.

Architecture

Simple, well-understood stack. A few configuration items worth hardening as we go, not rebuilding.

Our Approach

Sequenced by risk, not just by priority

  • Review first. Understand the system before changing it.
  • Fix what's visible. Reports remediation rebuilds trust early.
  • Build what matters most. Restocking, the operations team's top ask.
  • Test throughout, not at the end. Every phase ships with coverage for what it touched.
5 / 12
R4 — Architecture Review

Understand it, document it, harden it

  • Audit the stack, data flow, and full API surface.
  • Deliver current-state documentation suitable for IT handoff.
  • Flag configuration items worth addressing (e.g. access controls, data storage) as we go.
6 / 12
R1 — Reports Remediation

Bring Reports in line with the rest of the app

  • Migrate to the same patterns already used by every other view.
  • Add translation support, matching the system already in place elsewhere.
  • Remove leftover debug output; resolve filter and formatting inconsistencies.
  • Treat Meridian's 8 logged issues as a floor — full audit included.
7 / 12
R2 — Restocking Recommendations

From data to a purchasing decision

Stock levels + demand forecast + budget ceiling → recommended purchase orders.

  • New view built on data the system already has.
  • Completes purchase-order functionality the previous vendor started but didn't finish.
  • The centerpiece of this engagement for the operations team.
8 / 12
R3 — Automated Testing

A safety net that grows with every phase

  • No coverage exists today — this unblocks IT's ability to approve future changes.
  • Tests ship alongside each phase's work, not as a separate deliverable at the end.
  • Flows prioritized by risk, informed by the architecture review.
